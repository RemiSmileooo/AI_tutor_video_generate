大模型应用开发入门：从 Prompt 到 RAG

今天这节课，我们一起入门大模型应用开发。很多同学都用过 ChatGPT，但当我们真正要把大模型用到自己的产品里时，往往会遇到一系列问题：模型答非所问、信息过时、还会一本正经地胡说八道。这节课，我会带大家从最基础的 Prompt 讲起，一步步理解大模型应用开发的核心思路，最后落到一个非常实用的技术——检索增强生成，也就是 RAG。

首先，我们要理解什么是大语言模型。大语言模型，英文叫 Large Language Model，简称 LLM。它的本质是一个根据上文预测下一个词的概率模型。它通过在海量文本上训练，学会了语言的规律、世界的常识，以及一定的推理能力。需要强调的是，模型本身并不"知道"事实，它只是在预测最可能出现的文字。这也是它会"幻觉"的根本原因。理解这一点，对我们后面的设计非常关键。

接下来，我们讲第一个核心技能：Prompt 工程。所谓 Prompt，就是我们给模型的输入指令。同样一个任务，不同的写法，效果可能天差地别。写好 Prompt 有三个要点：第一，角色设定，明确告诉模型它扮演什么身份；第二，任务描述，把要做的事情说清楚，最好给出步骤；第三，输出格式，规定好返回的结构，比如用 JSON 或者分点列出。掌握这三点，你就能让模型的输出稳定很多。

但是，Prompt 工程也有它的天花板。它解决不了两个根本问题：一是模型的知识是有截止日期的，它不知道最新发生的事；二是模型不了解你私有的、内部的数据，比如公司的文档、产品手册。这时候，只靠改 Prompt 是没用的，我们需要给模型"外接知识"。

于是就引出了今天的重点：RAG，检索增强生成。它的核心思想其实很朴素：在让模型回答之前，先去知识库里检索出相关的资料，再把这些资料拼接到 Prompt 里，让模型基于这些资料来回答。这样模型就不再是凭空想象，而是"开卷考试"。RAG 通常分为四步：第一步，把文档切分成小块并转成向量存入向量数据库；第二步，把用户的问题也转成向量；第三步，在向量库里检索出最相似的几个文档块；第四步，把检索结果和问题一起交给大模型生成最终答案。

最后，我们做个总结。这节课我们建立了三个关键认知：第一，大模型的本质是预测下一个词，所以会有幻觉；第二，Prompt 工程是引导模型的基本功，但有天花板；第三，RAG 通过外接知识库，有效解决了知识过时和私有数据的问题，是当前大模型应用落地最主流的方案之一。希望大家课后能动手搭一个属于自己的小型 RAG 应用，把今天的内容真正用起来。
